# ウーパールーパーの餌になるWebサイト 🦎

顔写真を撮ると、**ウーパールーパーから見た「餌としての価値」** が各項目で診断されるWebサイトです。

> ウーパールーパーのためのサイトのはずなのに、利用するのは人間だけ。
> あなたは「餌」として評価されることで、その不思議なねじれを体験します。
> ── 「誰かのために作られたように見える」が、実際には誰の役にも立っていない「役に立たない機械」。

## 機能

- 📷 **カメラ撮影 / 画像アップロード**（`getUserMedia`、カメラが使えない端末はファイル選択にフォールバック）
- 🙂 **顔検出**（MediaPipe Tasks Vision）— 顔が写っていないと診断できません
- 📊 **餌診断** — 栄養価・捕食成功率・食べやすさ・好感度・本日のおすすめ度を **毎回ランダム** に評価
- 🖼️ **結果をシェア画像として書き出し**（端末で保存 / Web Share）
- 🔒 撮影画像は **端末内だけで処理** され、どこにも送信・保存されません

## ローカルで動かす

カメラは `https` か `localhost` でのみ動作します（`file://` 直開きは不可）。

```bash
# どれか
npx http-server -p 8000        # Node
python -m http.server 8000     # Python
```

→ ブラウザで <http://localhost:8000/> を開く。

## GitHub Pages で公開する

このリポジトリはそのまま公開できる静的サイトです（ビルド不要）。

1. GitHub の **Settings → Pages**
2. **Build and deployment → Source** を「Deploy from a branch」に
3. **Branch** を `main` / フォルダ `/ (root)` に設定して保存
4. 数分後 `https://<ユーザー名>.github.io/ika/` で公開されます

GitHub Pages は HTTPS で配信されるため、公開後はスマホのカメラでもそのまま診断できます。
`.nojekyll` を置いてあるので Jekyll の処理はスキップされます。

## 技術メモ

- 顔検出モデルと WASM は jsDelivr / Google の CDN から動的読み込み。CDN に到達できない場合は顔検出をスキップして診断を続行します（アプリは止まりません）。
- 診断結果は擬似乱数による生成で、再現性はありません（コンセプト通り「基準は不明・毎回変化」）。
